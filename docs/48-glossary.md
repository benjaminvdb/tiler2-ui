# Glossary

This document defines technical terms, acronyms, and project-specific terminology used throughout the documentation and codebase.

## A

### AI Message
A message generated by the AI assistant in response to user input. Contains the assistant's text response and optionally tool calls.

**See:** [35-message-format.md](/home/user/tiler2-ui/docs/35-message-format.md)

### API (Application Programming Interface)
Interface for communication between software components. In this project, refers to the LangGraph API for agent interactions.

### Artifact
Generated content (code, documents, data) displayed in the side panel. Created by the AI during conversation.

**See:** [13-artifacts.md](/home/user/tiler2-ui/docs/13-artifacts.md)

### Async/Await
JavaScript syntax for handling asynchronous operations. Makes asynchronous code look synchronous.

```typescript
async function fetchData() {
  const data = await fetch('/api/data');
  return data.json();
}
```

### Auth0
Authentication and authorization platform using OAuth2/OIDC protocols.

**See:** [16-authentication.md](/home/user/tiler2-ui/docs/16-authentication.md)

## B

### Base64
Binary-to-text encoding scheme. Used for embedding images in messages.

```typescript
const imageData = "data:image/png;base64,iVBORw0KGgo...";
```

### Breakpoint
1. (Debugging) Point where code execution pauses for inspection
2. (CSS) Screen width threshold for responsive design

### Bundle
Compiled JavaScript files served to the browser. Created by Vite during build process.

**See:** [24-build-process.md](/home/user/tiler2-ui/docs/24-build-process.md)

## C

### Callback
Function passed as argument to another function, executed later.

```typescript
setTimeout(() => console.log('Done'), 1000); // Arrow function callback
```

### Checkpoint
Snapshot of conversation state at a point in time. Enables "time travel" to explore different conversation paths.

**See:** [36-thread-schema.md](/home/user/tiler2-ui/docs/36-thread-schema.md)

### Client-Side
Code that runs in the user's browser (as opposed to server-side).

### Component
Reusable, self-contained piece of UI in React. Can be a function or class.

```typescript
function Button({ onClick, children }) {
  return <button onClick={onClick}>{children}</button>;
}
```

### CORS (Cross-Origin Resource Sharing)
Browser security feature that restricts web pages from making requests to different domains.

**See:** [43-common-issues.md](/home/user/tiler2-ui/docs/43-common-issues.md)

### CSP (Content Security Policy)
Security header that controls which resources browsers can load. Helps prevent XSS attacks.

**See:** [41-security.md](/home/user/tiler2-ui/docs/41-security.md)

### CSRF (Cross-Site Request Forgery)
Attack where unauthorized commands are transmitted from a trusted user. Prevented by state parameter in OAuth.

## D

### Dependency
External package/library the project relies on. Defined in `package.json`.

**See:** [46-dependencies.md](/home/user/tiler2-ui/docs/46-dependencies.md)

### DevTools
Browser developer tools for debugging (Console, Network, Elements, etc.).

**See:** [39-debugging.md](/home/user/tiler2-ui/docs/39-debugging.md)

### DOM (Document Object Model)
Tree structure representing HTML document in browser. React creates virtual DOM for efficiency.

## E

### Environment Variable
Configuration value stored outside code. Used for secrets and environment-specific settings.

**See:** [42-environment-variables.md](/home/user/tiler2-ui/docs/42-environment-variables.md)

### ESLint
Linting tool for JavaScript/TypeScript. Enforces code quality rules.

**See:** [38-code-quality.md](/home/user/tiler2-ui/docs/38-code-quality.md)

### ES2020
Version of ECMAScript (JavaScript standard) from 2020. Includes optional chaining, nullish coalescing, etc.

## F

### Fast Refresh
Vite/React feature that preserves component state during development when code changes.

### Feature Flag
Configuration toggle to enable/disable features. Example: `VITE_HIDE_TOOL_CALLS`.

## G

### Git
Version control system for tracking code changes.

### Graph
Directed graph representing AI workflow in LangGraph. Nodes are steps, edges are transitions.

**See:** [10-workflows.md](/home/user/tiler2-ui/docs/10-workflows.md)

## H

### HMR (Hot Module Replacement)
Development feature that updates modules without full page reload. Vite provides instant HMR.

### Hook
React function starting with "use" that lets you use state and lifecycle features in function components.

```typescript
const [count, setCount] = useState(0);
const ref = useRef(null);
useEffect(() => { /* ... */ }, []);
```

### Human Message
Message from the user. Can contain text, images, or files.

**See:** [35-message-format.md](/home/user/tiler2-ui/docs/35-message-format.md)

### Human-in-the-Loop
Workflow pattern where AI pauses and waits for human input before proceeding.

**See:** [12-human-in-loop.md](/home/user/tiler2-ui/docs/12-human-in-loop.md)

## I

### Interrupt
Point in AI workflow where execution pauses for user decision.

```typescript
interface InterruptItem {
  id: string;
  type: string;
  message?: string;
}
```

**See:** [12-human-in-loop.md](/home/user/tiler2-ui/docs/12-human-in-loop.md)

## J

### JSX (JavaScript XML)
Syntax extension for JavaScript that looks like HTML. Used in React components.

```typescript
const element = <div className="container">Hello</div>;
```

### JWT (JSON Web Token)
Compact token format for securely transmitting information. Used by Auth0 for authentication.

## K

### Knip
Tool for detecting unused files, exports, and dependencies.

**See:** [38-code-quality.md](/home/user/tiler2-ui/docs/38-code-quality.md)

## L

### LangChain
Framework for building applications with LLMs. Provides message standards and tools.

### LangGraph
Framework for building stateful, multi-step LLM workflows. Backend for this application.

**See:** [15-api-integration.md](/home/user/tiler2-ui/docs/15-api-integration.md)

### LLM (Large Language Model)
AI model trained on vast amounts of text. Examples: GPT-4, Claude, Llama.

### Linting
Process of checking code for errors and style issues using tools like ESLint.

## M

### Markdown
Lightweight markup language for formatted text. Used for message rendering.

```markdown
# Heading
**Bold** *italic* `code`
- List item
```

### Memoization
Optimization technique that caches function results. React provides `useMemo` and `React.memo`.

```typescript
const result = useMemo(() => expensiveOperation(), [deps]);
```

### Multimodal
Content containing multiple types (text, images, files) in single message.

**See:** [11-multimodal.md](/home/user/tiler2-ui/docs/11-multimodal.md)

## N

### NPM (Node Package Manager)
Package manager for JavaScript. This project uses pnpm instead.

### Node.js
JavaScript runtime for executing JavaScript outside the browser.

## O

### OAuth2
Industry-standard authorization protocol. Used by Auth0 for authentication.

### OIDC (OpenID Connect)
Authentication layer built on OAuth2. Provides user identity information.

## P

### PKCE (Proof Key for Code Exchange)
Security extension to OAuth2 for public clients (SPAs). Prevents authorization code interception.

**Pronunciation:** "pixy"

**See:** [16-authentication.md](/home/user/tiler2-ui/docs/16-authentication.md)

### pnpm
Fast, disk-efficient package manager for Node.js. Used in this project.

### Prettier
Opinionated code formatter. Ensures consistent code style.

**See:** [38-code-quality.md](/home/user/tiler2-ui/docs/38-code-quality.md)

### Props (Properties)

```typescript

  return <div>{name} is {age}</div>;

```

## R

### Radix UI

### React
JavaScript library for building user interfaces with components.

### Render

## S

### Sentry
Error tracking and performance monitoring service.

**See:** [26-monitoring.md](/home/user/tiler2-ui/docs/26-monitoring.md)

### SPA (Single-Page Application)
Web app that loads single HTML page and updates dynamically without full page reloads.

Server push technology where server sends updates to client over HTTP. Used for streaming.

**See:** [08-chat-system.md](/home/user/tiler2-ui/docs/08-chat-system.md)

Function that narrows TypeScript type based on runtime check.

```typescript

}
```

## Symbols

### => (Arrow Function)
Concise function syntax in JavaScript/TypeScript.

```typescript
const add = (a, b) => a + b;
const greet = name => `Hello ${name}`;
```

### ? (Optional Chaining)
Safely access nested properties that might be null/undefined.

```typescript
const name = user?.profile?.name; // undefined if user or profile is null
```

### ?? (Nullish Coalescing)
Returns right operand when left is null/undefined (but not other falsy values).

```typescript
const value = input ?? 'default'; // Uses 'default' only if input is null/undefined
const other = input || 'default'; // Uses 'default' for any falsy value (0, '', false)
```

### ... (Spread Operator)
Expands array/object into individual elements.

```typescript
const newArray = [...oldArray, newItem];
const newObject = { ...oldObject, newProp: value };
```
